## A. Унимодулярность

### A.1 Проблема в стандартной НКГ (контекст)

В программе Коннеса–Хамседдина унитарная группа конечной алгебры

```
A_F = ℂ ⊕ ℍ ⊕ M₃(ℂ)
```

равна **U(1) × SU(2) × U(3)**, а не точной калибровочной группе Стандартной
модели U(1) × SU(2) × SU(3). Переход U(3) → SU(3) требует наложения
**условия унимодулярности**: детерминант унитарного элемента из компоненты M₃(ℂ)
равен 1. Это условие добавляется *отдельно* и не выводится из общих аксиом НКГ
(Хамседдин–Коннес, 2007; Bochniak–Sitarz, 2022; Lizzi–Huggett–Menon, 2021).

Математически: гиперзарядный генератор U(1)_Y должен быть бесследовой
линейной комбинацией генераторов компонент A_F. В представлении фермионов
16-мерного мультиплета (одно поколение)

```
Y = diag(−1/3, −1/3, −1/3, 1, 1/3, 1/3, 1/3, 0, ...)   (в ≪лептон+кварк≫ базисе)
```

бесследовость Tr(Y) = 0 выполняется ровно тогда, когда набор гиперзарядов СМ
аномально свободен — это нетривиальное условие, автоматически выполняемое
спектром СМ.

### A.2 H4-производная калибровочная группа: структура

В Trinity S3AI калибровочная группа мотивируется через цепочку

```
H4  →  A4 (≅ SU(5))  →  SU(3)_C × SU(2)_L × U(1)_Y
```

с числом Коксетера h = 30 = 2 × 3 × 5, обеспечивающим три делителя,
ассоциированных с рангами SU(2), SU(3), SU(5) (доказано в GaugeOrigins.v).

Унитарная группа корневой системы H4 действует на 4-мерном пространстве
корней; натуральные «генераторы», появляющиеся из разложения h = 2 × 3 × 5,
не имеют матричного представления с определённым следом в смысле НКГ, пока
не задан явный фермионный мультиплет.

### A.3 Проверка бесследовости: три случая

**Случай 1. A4-подсистема (путь через SU(5))**

W(A4) = S₅, |Aut(A4)| = 240. В стандартном SU(5)-ГУТ гиперзарядный генератор
в фундаментальном **5** представлении имеет вид

```
Y = diag(2/3, 2/3, 2/3, −1, −1)  →  Tr(Y) = 2 − 2 = 0.
```

Это бесследовый генератор внутри A4 = su(5). **Вывод:** путь H4 → A4 → SU(5)
*автоматически* обеспечивает бесследовость гиперзарядного генератора,
поскольку A4-алгебра по определению состоит из бесследовых матриц.
Таким образом, для этого пути **условие унимодулярности выполнено автоматически**.

**Случай 2. Прямая проекция A2 × A2 → SU(3)_C × SU(2)_L**

Из G01_G02_origins.md: W(A2 × A2) ⋊ Z₂ ≅ SU(3)_C × SU(3)_L.
Обе алгебры A2 = su(3) состоят из бесследовых матриц. Следовательно,
любой генератор, порождённый из этой подсистемы, является бесследовым.
**Вывод:** унимодулярность выполнена для этого пути.

**Случай 3. U(1)_Y и числа Коксетера**

Проблема появляется там, где «ранговое» происхождение U(1)_Y не связывает
гиперзаряды конкретных фермионов. В Trinity S3AI числа {2, 12, 20, 30}
задают порядки периодичности, а не прямые матричные генераторы. Для явной
связи с U(1)_Y нужен выбор вложения U(1)_Y ↪ SU(5) с нормировкой
GUT (g' = g₁ · √(3/5)), что ровно совпадает со стандартной конструкцией
Джорджи–Глэшоу. Этот шаг **не выводится** из H4-данных, а постулируется
по аналогии.

### A.4 Честный вердикт по A

| Путь | Унимодулярность | Замечание |
|------|----------------|-----------|
| H4 → A4 → SU(5) → SU(3)×SU(2) | **ДА, автоматически** | A4 = su(5) по определению бесследова |
| H4 → A2×A2 → SU(3)×SU(2) | **ДА, автоматически** | A2 = su(3) бесследова |
| H4 → U(1)_Y (гиперзарядный генератор) | **НЕ ВЫВОДИТСЯ** | Требует выбора GUT-нормировки, как в SU(5) |

**Вердикт A:**  
Если Trinity S3AI идёт через SU(5)-путь (A4 ⊂ H4), то условие унимодулярности
выполнено автоматически, поскольку su(5)-алгебра по определению бесследова.
Однако само по себе H4 не порождает конкретный фермионный мультиплет с
фиксированными гиперзарядами — это требует дополнительного физического выбора.
Таким образом, в рамках Trinity S3AI унимодулярность **реализуется по наследованию
от A4 ⊂ H4**, но не является новым теоретическим результатом по сравнению с
стандартной SU(5)-ГУТ.

## B. σ-поле

### B.1 Что такое σ в стандартной НКГ

Согласно arXiv:1208.1030 (Хамседдин–Коннес, 2012):

> «Вещественное скалярное поле, сильно связанное с полем Хиггса, уже присутствовало
> в спектральной модели, но ошибочно игнорировалось в предыдущих вычислениях.»

Поле σ — это **нейтральный НКГ-синглет**: вещественное скалярное поле с
квантовыми числами B−L = 0. Оно возникает из сопряжения между компонентами
M₃(ℂ) (цветовой) и ℂ (лептонной) секции фермионного оператора Дирака D_F.
Технически σ появляется как флуктуация ≪скалярной связи≫ в секторе
(ν_R, ν_R) — правые нейтрино — в матрице Майорана внутри D_F.

В NCG-действии поле σ модифицирует β-функции УРГ для константы Хиггсового
самосцепления λ, аннулируя ИК-нестабильность λ в окне Λ_{SM} ≲ μ ≲ Λ_{унификация}
и восстанавливая согласие с m_H ≈ 125 ГэВ.

Ключевая структурная роль σ: оно возникает из **несиммершего (non-commutative)**
сектора, который отсутствует в чисто коммутативном пределе. Его появление
обусловлено тем, что алгебра A_F = ℂ ⊕ ℍ ⊕ M₃(ℂ) имеет ненулевой «межфакторный»
компонент оператора D_F.

### B.2 Есть ли аналог σ в 600-клетке?

**Анализ спектральной структуры H4:**

Оператор Дирака 600-клетки действует на 480-мерном пространстве (120 вершин × 4
спиновых компоненты). Его собственные значения группируются по симметриям H4.
В `SpectralAction600Cell.v` и `SpectralExtras.v` формализованы:
- N_vertices = 120 = 4h
- Spectral coefficient a4 = 1/(16φ) + φ³/8
- Vertex dominance: a4_vert > a4_curv

Для σ-аналога нужен **скалярный синглет** в разложении 600-клеточного Дирака.
Рассмотрим каждый критерий:

**Критерий 1. «Межфакторное» происхождение**

В стандартной НКГ σ возникает из связи между компонентами ℂ и M₃(ℂ) в A_F.
В Trinity S3AI единственная алгебра — спектральные данные 600-клетки; нет
двух «факторов» (ℂ и M₃(ℂ)), между которыми могло бы возникнуть σ-подобное
поле. H4-оператор Дирака имеет *единственный* спектральный источник —
геометрию корневой системы — без внутренней «двухфакторной» структуры A_F.

**Критерий 2. Правые нейтрино и масштаб Майорана**

В НКГ σ ∼ M_R масштабирует матрицу масс Майорана для правых нейтрино.
В Trinity S3AI NeutrinoOrigins.v содержит Axiom для Майоранового масштаба
(явно Admitted). H4 не порождает правых нейтрино из первых принципов:
их включение требует расширения фермионного мультиплета за пределы
«автоматически» возникающих представлений H4.

**Критерий 3. Скалярный синглет в спектре 600-клетки**

Собственные значения оператора Дирака 600-клетки (спектр H4) известны
из теории Коксетеровских групп: они имеют вид ±(e_i + e_j), где e_i —
показатели. Для H4: e = {1, 11, 19, 29}. Скалярный инвариант (l = 0)
в разложении по сферическим гармоникам 4D — это единственное **радиальное**
собственное значение. Пространство H4-инвариантных скалярных функций
порождается многочленами от четырёх основных инвариантов степеней {2, 12, 20, 30}.

Кандидат на σ-аналог: **инвариант степени 2** (d₁ = 2) — это квадрат радиуса
в 4D, который играет роль «синглета» при всех отражениях H4. Этот инвариант
существует и однозначно определён.

**Масштаб:**

```
m_σ_analog ≈ ℏ · (d₁ / d₄) · Λ_H4 = (2/30) · 1.5×10¹⁶ ГэВ ≈ 10¹⁵ ГэВ
```

Это масштаб Великого объединения — тот же порядок, что и m_σ в стандартной
НКГ (масштаб Майорана ~ 10¹⁴–10¹⁵ ГэВ). Численное совпадение есть.

**Однако:** этот «инвариант степени 2» — просто квадрат радиуса, то есть
постоянная на каждой орбите H4. Это не *динамическое* поле, которое могло бы
изменять β-функции УРГ. В стандартной НКГ σ является **динамическим скалярным
полем** с собственным кинетическим и потенциальным членами в действии.

### B.3 Boyle–Farnsworth-алгебра и σ

В работе Boyle–Farnsworth (2018) поле σ с зарядом B−L = 2 появляется из
неассоциативной надстройки супералгебры B = A ⊕ H. Для H4 нет известного
аналога такой надстройки в рецензируемой литературе.

### B.4 Следствие для m_H-предсказания

В Trinity S3AI ошибка m_H ≈ 6% (H01 предсказывает ≈ 125.20 ГэВ против
наблюдаемых ≈ 125.09 ГэВ по PDG 2024, ошибка ~0.09% в абсолютных единицах,
но ~6% в относительных, если рассматривать предыдущую версию формулы).

В НКГ σ фиксирует m_H, изменяя **β-функцию λ** при унификации. Для этого нужно:
1. λ(Λ) задано из спектральной модели
2. σ вносит вклад в β_λ через Yukawa-подобную связь y_σ ~ O(1)
3. Результирующее m_H = 125 ГэВ при μ = m_Z

Без динамического σ-аналога в H4/600-клетке этот механизм **не воспроизводим**.
H01 = 4φ³e² — это прямая формула без RGE-коррекций. Точность 0.09% означает,
что формула H01 уже хорошо согласуется с данными; «6%-я ошибка» относилась к
предыдущей версии. Тем не менее, без σ-подобной динамики нет механизма, который
мог бы «зафиксировать» это значение из первых принципов.

### B.5 Честный вердикт по B

| Критерий | Статус в H4 | Комментарий |
|----------|------------|-------------|
| Нейтральный синглет в спектре | Частично — d₁=2 инвариант | Не динамическое поле |
| Межфакторное происхождение | **НЕТ** | H4 не имеет «ℂ + M₃(ℂ)»-структуры |
| Правосторонние нейтрино | **НЕ ВЫВОДИТСЯ** | Admitted в NeutrinoOrigins.v |
| Масштаб ~ 10¹⁵ ГэВ | Численное совпадение | d₁/d₄ · Λ_H4 ~ правильный порядок |
| Модификация β_λ | **ОТСУТСТВУЕТ** | Нет динамического действия для σ |
| Фиксация m_H методами RGE | **НЕДОСТИЖИМО** без σ | Честное ограничение |

**Вердикт B:**  
H4/600-клеточная структура **не порождает** естественного аналога σ-поля
в смысле Хамседдина–Коннеса. Инвариант степени d₁ = 2 является геометрическим
синглетом, но не динамическим скалярным полем с действием. Следовательно,
6%-я ошибка в предсказании m_H (в случае, если она присутствует в конкретной
версии H01) **не может быть исправлена методами, аналогичными σ Коннеса** в
рамках текущей Trinity S3AI конструкции. Это честный отрицательный результат.

## Выводы для HiggsOrigins

1. **H01** (m_H = 4φ³e² ≈ 125.20 ГэВ) достигает точности класса V без
   каких-либо RGE-коррекций. Если при обновлённых данных PDG погрешность
   составляет ~0.09%, она не требует σ-коррекции.

2. Если интерпретировать результат как «нужно ~6% улучшение», то
   **метод σ-поля НКГ неприменим** к H4, поскольку H4 не имеет
   двухфакторной алгебраической структуры, из которой σ возникает в НКГ.

3. Альтернативный путь к улучшению H01: поиск **следующего порядка** в
   спектральном разложении 600-клетки (вклады a₂ или a₆), что является
   отдельной технической задачей.

4. Условие **унимодулярности** в H4-подходе хорошо: путь H4 → A4 → SU(5) → SM
   автоматически даёт бесследовые генераторы. Это поддерживает самосогласованность
   калибровочного сектора Trinity S3AI.
