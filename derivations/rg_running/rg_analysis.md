## 1. Точная формулировка двух Admitted теорем

### Теорема 2: `alpha_from_H4`

```coq
Theorem alpha_from_H4 :
  Rabs (alpha_inv_at_mZ - trinity_alpha_inv) / trinity_alpha_inv < 1/100.
```

Где:
- `alpha_inv_at_mZ := alpha_i_inv m_Z b1 + alpha_i_inv m_Z b2`
- `alpha_i_inv mu b_i := gU2inv + (b_i / (4 * PI * PI)) * ln (Lambda_H4 / mu)`
- `gU2inv ∈ [1/26, 1/22]` (аксиома `gU2inv_window`)
- `trinity_alpha_inv := 36 * phi * e² / PI ≈ 137.003`

### Теорема 3: `alpha_s_from_H4`

```coq
Theorem alpha_s_from_H4 :
  Rabs (alpha_s m_Z - trinity_alpha_s) / trinity_alpha_s < 1/50.
```

Где:
- `alpha_s mu := 1 / alpha_i_inv mu b3`
- `trinity_alpha_s := (sqrt 5 - 2) / 2 ≈ 0.11803`

## 2. Почему теоремы физически некорректны (некорректная постановка)

### 2.1 Проблема нормировки гиперзаряда (фактор 5/3)

Величина `alpha_inv_at_mZ` определена как:

```
alpha_inv_at_mZ = alpha_i_inv(m_Z, b1) + alpha_i_inv(m_Z, b2)
```

Это **не** является 1/α_em(m_Z) ни в каком стандартном соглашении. Физически корректная формула Вайнберга в ГЕТ-нормировке SU(5):

```
1/α_em(m_Z) = (5/3) · 1/α₁(m_Z) + 1/α₂(m_Z)
```

Множитель **5/3** (нормировка гиперзаряда) обязателен потому, что в СМ гиперзарядная связь g' связана с ГЕТ-нормированной связью g₁ как:

```
g' = sqrt(3/5) · g₁
```

то есть `α_Y = g'²/(4π) = (3/5) · α₁`, откуда `1/α_Y = (5/3) · 1/α₁`.

### 2.2 Несоответствие численных значений

Численный расчёт при `gU2inv = 1/24`, `ln(Λ_H4/m_Z) ≈ 32.73`:

| Величина | Значение в коде | Физическое значение (PDG 2023) |
|----------|----------------|-------------------------------|
| `alpha_i_inv(m_Z, b1)` | ≈ 3.441 | Нестандартная единица |
| `alpha_i_inv(m_Z, b2)` | ≈ -2.584 | Нестандартная единица |
| `alpha_inv_at_mZ` | ≈ 0.857 | 1/α_em(m_Z) ≈ 127.951 |
| `trinity_alpha_inv` | ≈ 137.003 | α⁻¹(0) = 137.036 (Томсон) |

Отношение: `alpha_inv_at_mZ / trinity_alpha_inv ≈ 0.006` — отклонение составляет ~99.4%, а не <1%.

**Физическая несоответствие** носит принципиальный характер:
- `alpha_inv_at_mZ ~ O(1)` (безразмерная, но не 1/α_em)
- `trinity_alpha_inv ~ 137` (формула Тринити претендует на значение ~137)
- Разница — порядок величины ~160, а не проценты

### 2.3 Конвенционная проблема формулы бегущего

Код использует:
```
alpha_i_inv(mu, b_i) = gU2inv + (b_i / 4π²) · ln(Λ/μ)
```

Стандартная одно-петлевая формула Джорджи–Куинн–Вайнберга (ГКВ):
```
1/α_i(μ) = 1/α_GUT + (b_i / 2π) · ln(Λ/μ)
```

Эти формулы **не идентичны**: коэффициент `1/(4π²)` в коде против `1/(2π)` в ГКВ отличается на множитель `π/2`. Это означает, что `alpha_i_inv` в коде не является стандартным `1/α_i`. Фактически, с учётом соотношения α = g²/(4π):

```
1/g_i²(μ) = gU2inv + (b_i / 4π²) · ln(Λ/μ)   [формула кода]
```

совпадает со стандартной только если `gU2inv = 1/g_GUT²` — но тогда `gU2inv ≈ 1/24` предполагает `g_GUT² = 24`, то есть `α_GUT = g²/(4π) ≈ 24/(4π) ≈ 1.91`. Это не физическое значение.

### 2.4 Проблема `alpha_s`

```coq
Definition alpha_s (mu : R) : R := 1 / alpha_i_inv mu b3.
```

Это `1 / (1/g3²)` = `g3²`. Физический `α_s = g3²/(4π)`. Отсутствует множитель `1/(4π)`. Кроме того, при `gU2inv = 1/24` получаем `alpha_i_inv(m_Z, b3) ≈ -5.76 < 0` — отрицательное! (Код это явно обходит аксиомой `alpha_run_window`.) Поэтому `alpha_s(m_Z)` из кода даст `1/(-5.76) ≈ -0.174` — не 0.118.

**Вывод:** Обе теоремы физически некорректно поставлены. Закрыть их в текущей форме нельзя — они математически ложны при данных аксиомах.

## 3. Предлагаемая переформулировка (ДОКАЗУЕМАЯ)

### 3.1 Для `alpha_from_H4`

**Проблема:** `alpha_inv_at_mZ` не равно 1/α_em(m_Z).

**Корректная переформулировка** (на уровне математики, без физических входных данных):

> **Лемма `hypercharge_normalization_identity`:**  
> Для любого значения обратных констант связи α₁⁻¹ и α₂⁻¹,  
> физический `1/α_em = (5/3)·α₁⁻¹ + α₂⁻¹`, а не `α₁⁻¹ + α₂⁻¹`.  
> Доказуемо из алгебры: `(5/3)·x + y ≠ x + y` для разумных x, y > 0.

**Доказуемое Coq-утверждение** (`RGRunningExtras.v`):
```coq
(* Алгебраическая лемма о факторе 5/3 *)
Lemma alpha_em_formula_correction :
  forall a1_inv a2_inv : R,
    a1_inv > 0 -> a2_inv > 0 ->
    (5/3) * a1_inv + a2_inv = a1_inv + a2_inv + (2/3) * a1_inv.
(* Тривиально из кольцевой арифметики *)
```

Более содержательная версия:

```coq
(* Физически корректная формула EM-связи (алгебраическое тождество) *)
Definition alpha_em_inv_physical (a1_inv a2_inv : R) : R :=
  (5/3) * a1_inv + a2_inv.

Definition alpha_em_inv_code (a1_inv a2_inv : R) : R :=
  a1_inv + a2_inv.

Lemma gut_correction_factor :
  forall a1_inv a2_inv : R,
    a1_inv > 0 ->
    alpha_em_inv_physical a1_inv a2_inv > alpha_em_inv_code a1_inv a2_inv.
```

**Для закрытия `alpha_from_H4` потребуется:**
1. Переопределить `alpha_inv_at_mZ` с множителем 5/3
2. Добавить правильный множитель 4π для перехода от g² к α
3. Ввести физическую граничную аксиому в терминах стандартного `α_GUT = 1/24`
4. Убедиться, что `trinity_alpha_inv ≈ 127.95` (PDG), а не 137

Поскольку `trinity_alpha_inv ≈ 137.003` (формула Тринити), а PDG значение `1/α_em(m_Z) ≈ 127.951`, речь идёт о **разных вещах**: 137.036 — это низкоэнергетическая константа α⁻¹(0) (томсоновский предел), а 127.95 — это α⁻¹(m_Z) (бегущая при m_Z). Переформулировка также требует уточнения, **которое** из этих значений проверяется.

### 3.2 Для `alpha_s_from_H4`

**Корректная переформулировка:**

> **Лемма `alpha_s_definition_gap`:**  
> Код вычисляет `alpha_s(m_Z) = 1/alpha_i_inv(m_Z, b3)` — это g₃², но физический  
> α_s = g₃²/(4π). Таким образом, `alpha_s_code = 4π · alpha_s_physical`.

**Доказуемое Coq-утверждение:**
```coq
(* Связь между кодовым и физическим alpha_s *)
Lemma alpha_s_normalization :
  forall g3_inv_sq : R,
    g3_inv_sq > 0 ->
    1 / g3_inv_sq = 4 * PI * (1 / (4 * PI * g3_inv_sq)).
```

## 4. Честная оценка

### Что доказано в `RGRunning.v`:
- `H4_unification_scale`: **Доказано (Qed)** — тривиально из определения, что все связи совпадают в точке Λ_H4.
- `alpha_i_inv_pos_at_mZ`: **Доказано (Qed)** через аксиомы `gU2inv_window` и `alpha_run_window`.

### Что НЕ может быть доказано в текущей постановке:
1. **`alpha_from_H4`** — математически ложно при данных определениях. `alpha_inv_at_mZ ≈ 0.86` никогда не будет в 1% от `trinity_alpha_inv ≈ 137`. Расстояние ~160 единиц, а не <1%.

2. **`alpha_s_from_H4`** — математически ложно при данных определениях. `alpha_s(m_Z) = 1/alpha_i_inv(m_Z, b3) ≈ -0.17` (отрицательное!), тогда как `trinity_alpha_s ≈ 0.118`. Даже с аксиомой `alpha_run_window` (которая гарантирует `alpha_i_inv(m_Z, b3) > 0`), числовое значение не попадает в 2%.

### Что доказывается в `RGRunningExtras.v`:
1. **Алгебраическое тождество** гиперзарядного нормирования: `(5/3)·a₁⁻¹ + a₂⁻¹` vs `a₁⁻¹ + a₂⁻¹` (фактор коррекции — `(2/3)·a₁⁻¹`)
2. **Неравенство**: физически корректная формула всегда больше кодовой при положительных аргументах
3. **Формульная связь**: `alpha_s_physical = alpha_s_code / (4·π)`
4. **Числовые леммы**: вычисление коэффициента `5b₁/3 + b₂ = 11/3`, `b₁ + b₂ = 14/15`
5. **Граница для физической формулы**: при данной аксиоме `gU2inv_window`, корректно определённая `physical_alpha_em_inv` ограничена снизу

**Вывод:** Оба Admitted закрыть нельзя без:
- Переопределения `alpha_inv_at_mZ` с множителем 5/3 и фактором 4π
- Изменения `trinity_alpha_inv` на значение ~127.95 (или явного указания, что 137 — это низкоэнергетический предел)
- Физической аксиомы, связывающей `gU2inv` со стандартным `α_GUT`

Файл `RGRunningExtras.v` содержит **математически строгие** (Qed) леммы о нормировочных факторах и предлагает корректную переформулировку. Сами теоремы `alpha_from_H4` и `alpha_s_from_H4` в `RGRunning.v` остаются Admitted — честно и правомерно.
