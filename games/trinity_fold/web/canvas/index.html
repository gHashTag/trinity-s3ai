GOLDEN BRIDGE — Rust canvas UI (ring 4)

Build a structural bridge between data and geometry — without collapsing it. Pure-Rust render model + Canvas2D shell via wasm-bindgen. The game state lives in ring4_canvas::state::AppState; JavaScript only forwards mouse/key events and translates render primitives into draw calls.

Honesty notice. GOLDEN BRIDGE is a hypothesis-discovery puzzle, not a proven Theory of Everything. A high "bridge strength" only means the board passes more of the illustrative consistency checks than alternatives. Every tile is gated by a claim status; the worst such status is rendered next to the total. A falsified tile collapses the bridge regardless of the rest.

The live UI is a WebAssembly module compiled from crates/ring4_canvas. It is intentionally not checked into the repository (the bundle is build output, and shipping it would defeat the Rust-as-source-of-truth contract documented in docs/CANVAS.md). When the bundle is missing — typical for fresh clones and sandboxed previews — this page falls back to the static SVG below.

The SVG is generated by the same ring4_canvas::render::layout function the wasm shell uses, via: 

cd games/trinity_fold
cargo run -p ring4_canvas --example snapshot_svg -- \
  web/canvas/snapshot.svg

To enable the interactive build:

cd games/trinity_fold
wasm-pack build crates/ring4_canvas --target web --features wasm \
  --out-dir ../../web/canvas/pkg
