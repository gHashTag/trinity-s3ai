Trinity Fold — Rust canvas UI (ring 4)

Pure-Rust render model + Canvas2D shell via wasm-bindgen. The game state lives in ring4_canvas::state::AppState; JavaScript only forwards mouse/key events and translates render primitives into draw calls.

Honesty notice. Trinity Fold is a puzzle prototype, not a Theory of Everything. A high score only means the board passes more of the illustrative consistency checks than alternatives. Every tile is gated by a claim status; the worst such status is rendered next to the total.
WASM bundle missing. Build it once with: 
cd games/trinity_fold
wasm-pack build crates/ring4_canvas --target web --features wasm \
  --out-dir ../../web/canvas/pkg
See docs/CANVAS.md for full build instructions and the rationale for keeping the bundle out of version control.
